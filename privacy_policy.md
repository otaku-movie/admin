## 2. 我们收集的信息

我们仅在为您提供服务所必要的范围内，收集并处理下列信息。

### 2.1 您主动提供的信息

| 场景 | 信息项 | 是否必要 |
|---|---|---|
| 邮箱注册 | 邮箱、密码（加密存储）、昵称 | 必要 |
| Google / Apple 登录 | OpenID（`sub`）、邮箱、昵称、头像 URL | 必要 |
| 完善资料 | 头像、昵称、性别、生日 | 可选 |
| 下单 / 取票 | 收件信息、电影、场次、座位、订单金额 | 必要 |
| 评论 / 反馈 | 文本、图片、视频 | 可选 |

### 2.2 您使用本服务时我们自动收集的信息

| 类型 | 信息项 | 目的 |
|---|---|---|
| 设备信息 | 设备型号、操作系统及版本、`deviceId`（由 App 本地生成）、应用版本 | 安全风控、Refresh Token 绑定、兼容性适配 |
| 网络信息 | IP 地址、连接类型 | 防欺诈、接口路由 |
| 日志信息 | 接口访问时间、关键操作记录 | 故障排查、产品优化（仅服务端日志，**当前版本未接入第三方崩溃监控**）|
| 位置信息（**仅在您授权后**）| 设备定位经纬度，由 `geolocator` 与系统反向地理编码（`geocoding`）解析为城市/地区 | 推荐附近影院、就近选座 |
| 相册 / 相机内容（**仅在您主动选择后**）| 您选择上传或拍摄的图片，经 `image_picker` 选取、`image_editor` 在本地裁剪后上传至本平台 | 设置头像、提交反馈、上传评论图片 |
| 本地存储 | `accessToken` / `refreshToken` / `deviceId`（通过 `flutter_secure_storage` 加密保存）、语言与界面偏好（通过 `shared_preferences` 保存）| 维持登录、记住偏好；仅保存在本设备，不上传第三方 |

> 当前 App **尚未接入** Firebase Cloud Messaging / APNs 等推送 SDK，故未收集 FCM / APNs Token；后续若接入，将通过更新本协议与《第三方 SDK 清单》并取得您的同意。

### 2.3 我们不会收集的信息

- 完整的银行卡号、CVV、有效期等支付敏感信息**不经过我们的服务器**，未来接入支付时将由具备资质的第三方支付机构直接处理。
- 我们**不会**遍历您的相册、通讯录、短信或剪贴板；上传图片仅限您主动选择的那张。

## 3. 我们如何使用您的信息

我们仅在以下目的范围内使用您的个人信息：

1. 完成账号注册、登录、身份验证；
2. 履行电影票订单：包括下单、支付撮合、出票、入场核验、退改签、客服；
3. 安全保障：账号防盗、防止黄牛与刷单、风控审计；
4. 服务改进：基于**匿名化或去标识化**数据进行统计与产品优化；
5. 营销推送（仅在您未关闭推送的情形下）；
6. 履行法律法规规定的义务。

## 4. 信息共享、委托处理与公开披露

我们承诺**不会向无关第三方出售您的个人信息**。仅在下列情形下共享或委托处理：

### 4.1 与影院 / 院线共享

为完成订单与入场核验，我们会向相应影院共享：订单号、场次、座位、电子取票码及必要的姓名/手机后四位等。

### 4.2 与支付机构共享（**当前版本尚未上线在线支付**）

未来接入在线支付后，为完成支付、对账、退款，我们将向第三方支付机构共享必要信息（订单号、金额、币种、支付方式标识）。具体对接方上线前会更新本协议与《第三方 SDK 清单》。

### 4.3 委托处理（受托方按我们指示处理）

- 云服务：{{CLOUD_PROVIDERS}}
- 客服 / 邮件 / 短信：{{COMMS_PROVIDERS}}

我们与上述受托方签订数据处理协议，限制其仅在约定目的内处理数据，并要求其采取等同保护措施。

> 当前版本**尚未接入**第三方推送（FCM/APNs）、错误监控（Sentry 等）、统计分析（Firebase Analytics 等）。后续接入时将更新本协议并请求您的同意。

### 4.4 第三方登录

当您使用 Google / Apple 登录时，相关第三方会获知您使用本服务的事实并向我们返回必要的账号标识、邮箱、昵称、头像。该过程同时受第三方隐私政策约束（详见《第三方 SDK 清单》）。

### 4.5 法律披露

在法律法规、司法机关、行政机关合法要求的情形下，我们将依法披露相关信息，并尽合理努力通知您（法律禁止通知的除外）。

6.1 我们将您的个人信息存储在 {{STORAGE_LOCATIONS}} 的服务器，遵循当地适用法律的安全要求。
